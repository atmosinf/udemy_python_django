**OK, so let's discuss line by line what's actually happening here.
First, we had to import HTP response object from the Jango HTP module and then each view for this application
is going to exist within that views that py file as its own individual function.
And in this instance, we just created one view called index with that index function.
And each view is also going to take in at least one argument and for that HTTP request object since
it lives inside that Django HTP module by convention, we usually call this the request.
But again, you can actually just call this whatever you want.
This could be a jelly doughnut as a variable.
It really wouldn't matter.
But by convention we call it request.
And you'll notice that regardless of what is passed in to this function, it returns to TPE response
and then this text, Hello World.
So each view must return in response object.
And this very simple response object just is going to take in the string parameter representing the
content of the page.**<br><br>
